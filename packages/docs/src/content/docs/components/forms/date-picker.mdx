DatePicker composes [Popover](/components/feedback/popover/), [Calendar](/components/forms/calendar/), and a button-styled trigger into a ready-to-use date input. Two variants ship: `DatePicker` for a single date and `DateRangePicker` for a start–end range with a two-month preview.

## Import

```tsx
import { DatePicker, DateRangePicker } from '@nim-ui/components';
```

## Single Date

DatePicker supports both controlled (`value` + `onChange`) and uncontrolled (`defaultValue`) modes. The trigger shows the formatted date via [date-fns](https://date-fns.org/).

```tsx
import { useState } from 'react';
import { DatePicker } from '@nim-ui/components';

export function BookingForm() {
  const [date, setDate] = useState<Date | undefined>();

  return (
    <DatePicker
      value={date}
      onChange={setDate}
      placeholder="Pick a date"
    />
  );
}
```

## Custom Format

Pass any [date-fns format string](https://date-fns.org/docs/format) to control how the selected date is displayed in the trigger.

```tsx
<DatePicker value={date} onChange={setDate} format="yyyy-MM-dd" />
<DatePicker value={date} onChange={setDate} format="MMM d, yyyy" />
<DatePicker value={date} onChange={setDate} format="PPP" /> {/* default */}
```

## Range Picker

`DateRangePicker` renders two months side by side and returns a `DateRange` (`{ from, to }`) object.

```tsx
import { useState } from 'react';
import { DateRangePicker } from '@nim-ui/components';
import type { DateRange } from 'react-day-picker';

export function StayRange() {
  const [range, setRange] = useState<DateRange | undefined>();

  return (
    <DateRangePicker
      value={range}
      onChange={setRange}
      placeholder="Check-in — Check-out"
    />
  );
}
```

## Form Integration

DatePicker renders a hidden `<input type="hidden" name={name}>` with the ISO string value whenever `name` is provided — so it works with native form submissions and libraries like `react-hook-form` via `Controller`. Pass an `id` to wire it up with `FormField`:

```tsx
import { FormField, DatePicker } from '@nim-ui/components';

<FormField label="Event date" htmlFor="event-date">
  <DatePicker id="event-date" name="eventDate" />
</FormField>
```

## Uncontrolled

Skip `value` / `onChange` and pass `defaultValue` to let the component manage its own state.

```tsx
<DatePicker defaultValue={new Date()} />
```

## Disabled

```tsx
<DatePicker value={date} onChange={setDate} disabled />
```

## Accessibility

- Trigger is a button — fully keyboard-focusable with the same focus ring as other nim-ui controls
- Popover manages focus: opens to the calendar grid, returns focus to the trigger on close
- Pass an `id` and wire it to a `<label htmlFor>` (or use `FormField`) to announce the field to screen readers
- Calendar inherits the full keyboard support documented in [Calendar](/components/forms/calendar/)
- Hidden input (`name` prop) ensures the date is submitted natively without losing semantic meaning
