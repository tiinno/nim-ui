The Spacer component creates visual separation between elements using either fixed sizes or flexible (flex: 1) spacing. It is particularly useful inside Flex or Stack containers to push items apart without relying on margin or justify-content. The component includes `aria-hidden="true"` by default since it is a purely decorative element.

## Import

```tsx
import { Spacer } from '@tiinno-ui/components';
```

## Flexible Spacer

The `flex` prop applies `flex: 1` to the spacer, causing it to expand and fill all available space. This is useful for pushing items to opposite ends of a flex container.

<ComponentPreview title="Flex Spacer">
  <Flex client:load className="w-full bg-neutral-50 dark:bg-neutral-800 p-2 rounded">
    <div className="bg-green-100 dark:bg-green-900 p-3 rounded text-xs">Left</div>
    <Spacer client:load flex />
    <div className="bg-green-100 dark:bg-green-900 p-3 rounded text-xs">Right</div>
  </Flex>

## Usage Examples

### Header with Spacer

```tsx
function Header() {
  return (
    <Flex align="center">
      <img src="/logo.svg" alt="Logo" />
      <Spacer flex />
      <nav>
        <Flex gap="md">
          <a href="/about">About</a>
          <a href="/contact">Contact</a>
        </Flex>
      </nav>
      <Spacer size="md" />
      <Button variant="primary">Sign In</Button>
    </Flex>
  );
}
```

### Section Separator

```tsx
function PageContent() {
  return (
    <Stack spacing="none">
      <section>First section content</section>
      <Spacer size="2xl" />
      <section>Second section content</section>
      <Spacer size="xl" />
      <section>Third section content</section>
    </Stack>
  );
}
```

### Card Footer Actions

```tsx
function CardWithActions() {
  return (
    <Card>
      <CardContent>Card content here</CardContent>
      <CardFooter>
        <Button variant="ghost">Cancel</Button>
        <Spacer flex />
        <Button variant="primary">Save</Button>
      </CardFooter>
    </Card>
  );
}
```

## Accessibility

The Spacer component is a purely decorative element and includes `aria-hidden="true"` by default, ensuring it is invisible to screen readers and assistive technologies. No additional accessibility considerations are needed.
