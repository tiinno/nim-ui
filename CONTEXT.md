# Nim UI Context

Nim UI is a React UI kit for operational software. Its language prioritizes dashboard and backoffice workflows, with commerce treated as a supporting surface.

## Language

**Premium Operations Minimal**:
A refined visual direction for operational interfaces that favors calm hierarchy, restrained color, soft depth, and dense-but-readable workspace composition.
_Avoid_: Colorful SaaS, marketing dashboard, decorative admin UI

**Ink + Muted Steel Accent**:
The preferred color posture for Nim UI: graphite and ink carry primary actions, while a subdued steel blue-gray supports focus, selection, links, and quiet interactive emphasis.
_Avoid_: Bright blue primary, multi-accent palettes, decorative gradients

**Compact Premium Density**:
The preferred spatial posture for Nim UI: dense enough for backoffice scanning, with stable spacing, 8px component radii, quiet borders, and soft layered shadow.
_Avoid_: Airy landing-page spacing, oversized cards, pill-heavy controls

**Backoffice**:
The primary product context for Nim UI: internal tools used to review records, manage queues, inspect operational state, and act on workflow exceptions.
_Avoid_: Generic dashboard, admin panel

**Commerce Operations**:
The secondary product context for Nim UI: commerce-related operational screens such as orders, carts, fulfillment, pricing, and inventory work.
_Avoid_: Storefront, ecommerce landing page

## Example Dialogue

Developer: Should this component feel like a colorful SaaS analytics widget?

Domain expert: No. It should follow Premium Operations Minimal: quiet hierarchy, restrained color, and enough density for backoffice operators to scan and act.

Developer: Should the primary button use a bright blue brand color?

Domain expert: No. Use Ink + Muted Steel Accent so actions feel calm, premium, and operational.

Developer: Should cards feel spacious and promotional?

Domain expert: No. Use Compact Premium Density so operational screens stay scannable and composed.

Developer: Is the product card the main use case?

Domain expert: No. Commerce Operations support matters, but Backoffice remains the primary context.
